  <action>
First, inventory every inline `<img>` tag with its current alt text across all pages. Use grep to find them all:
```bash
grep -rn 'alt="[^"]*"' app/ --include="*.tsx" | grep '<img'
```

For each image, evaluate the alt text quality:
- **HIGH (keep as-is):** 20+ chars, describes what image shows, includes location and/or service context, reads naturally. Example: "5 Star Commercial Roofing team in Amarillo Texas - Professional roofing contractors and crew"
- **MEDIUM (rewrite):** Has some context but template-like or missing key details. Example: "Metal roof systems for Bushland TX" -- needs more description of what the image actually shows.
- **LOW (rewrite):** Under 20 chars or generic label. Example: "Metal Roofing", "EPDM Rubber Roofing", "Commercial Roofing"

**Alt text formula (not a rigid template -- vary the structure):**
`{What the image shows} - {service/material context} in {City}, TX | 5 Star Roofing`

Guidelines:
- **Describe what the IMAGE shows**, not what the page is about. If the image is a completed metal roof, say so: "Standing seam metal roof installation on a commercial building"
- **Include ONE city name** -- use the page's city. If no city (hub/services page), use "West Texas" or "Texas Panhandle"
- **Include ONE service keyword** -- match the page topic (hail damage repair, metal roofing, TPO installation, etc.)
- **Include brand** "5 Star Roofing" where it fits naturally, not forced into every alt
- **Length: 40-125 characters** -- descriptive but not keyword-stuffed
- **Vary structure** -- do NOT use identical template across all pages. Each alt should read uniquely.

**Specific rewrites for the most common generic alts:**

| Current Alt | Page Context | Rewrite To |
|-------------|-------------|------------|
| "Metal Roofing" | metal-roofing pages | "Standing seam metal roof installed on [City] TX home - 5 Star Roofing" |
| "EPDM Rubber Roofing" | commercial pages | "EPDM rubber membrane roof system on commercial building in [City] TX" |
| "Commercial Roofing" | commercial pages | "Commercial flat roof replacement project by 5 Star Roofing in [City] TX" |
| "Free Roof Inspections" | services page | "5 Star Roofing inspector examining shingle condition on Texas Panhandle home" |
| "Hail Damage Roof Repair" | services page | "Hail-damaged asphalt shingles showing impact marks requiring repair" |
| "Asphalt Shingle Roofing" | services/residential | "New asphalt shingle roof installation with clean ridge line in [City] TX" |
| "Insurance Claims Help" | services page | "5 Star Roofing team documenting storm damage for insurance claim" |
| "TPO Commercial Roofing" | TPO/commercial | "White TPO membrane being welded on flat commercial roof in [City] TX" |

These are starting points -- adapt based on the actual image (R2 URL gives clues: `completed-amarillo-4` = completed project in Amarillo, `hail-damage-borger-12` = hail damage in Borger, etc.).

**For gallery page images:** The gallery already has `caption` properties -- use those as the basis for alt text but make them more descriptive. Current captions like "Commercial TPO installation - Pampa, TX" are adequate but can be enriched slightly.

**Do NOT change:**
- Alt text that already scores HIGH quality (descriptive, 40+ chars, contextual)
- The `alt` on decorative images if they correctly use `alt=""`
- Any image `src` URLs (those were fixed in plan 09-01)

<task type="auto">
  <name>Task 2: Create alt text audit script and validate</name>
  <files>scripts/audit-alt-text.js</files>
  <action>
Create `scripts/audit-alt-text.js` that scans built HTML in `out/` and:

1. Finds every `<img>` tag in all HTML files
2. For each image, extracts `alt` and `src` attributes
3. Flags issues:
   - **Missing alt:** img tag has no alt attribute at all
   - **Empty alt on non-decorative:** alt="" on images with meaningful src (R2 URLs are content images, not decorative)
   - **Too short:** alt text under 20 characters (unless empty decorative)
   - **Too long:** alt text over 150 characters (keyword stuffing risk)
   - **Generic:** alt text matches known generic patterns: exactly "Metal Roofing", "Commercial Roofing", "EPDM Rubber Roofing", "TPO Commercial Roofing", "Asphalt Shingle Roofing", "Free Roof Inspections", "Insurance Claims Help", "Hail Damage Roof Repair"
   - **Duplicate on same page:** two images on the same page with identical alt text
4. Reports per-page and summary:
   - Total images audited
   - Issues by category (missing, empty, short, long, generic, duplicate)
   - Pages with issues listed
   - PASS/FAIL (PASS = zero missing, zero generic, zero same-page duplicates)

Use built-in Node.js only. Parse HTML with regex (sufficient for img tag extraction from static HTML).

Exit code: 0 on PASS, 1 on FAIL.

Run: `node scripts/audit-alt-text.js`

Fix any issues found and re-run until passing.
  </action>

3. All flagged images have been addressed
