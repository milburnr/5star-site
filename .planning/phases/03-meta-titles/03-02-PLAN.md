  <action>
Create `scripts/fix-meta-descriptions.js` following the established pattern from `scripts/standardize-nap.js`. The script must:

1. **File discovery:** Recursively find all `page.tsx` files under `app/`.
2. **Metadata isolation:** Extract only the `export const metadata: Metadata = { ... };` block. All transformations apply ONLY within this block.
3. **Description extraction:** Match `description: "value"` or `description: 'value'` within the metadata block. Handle both quote types. When reading single-quoted descriptions, be aware of apostrophes (e.g., "West Texas's") — match the outer quotes, not internal apostrophes.

**Fix rules (apply in this order):**
- **Capitalize first letter:** If description starts with a lowercase letter, capitalize it. (34 pages)
- **Remove double commas:** Replace ",," with "," anywhere in description. (Research says 0 found currently, but include as safety net.)
- **Rewrite "Looking for..." descriptions:** Replace with outcome-focused description per content-voice.md. For `/amarillos-best-roofer/`, rewrite to something like: `"Amarillo's highest-rated roofer for hail damage, storm repair, and full replacements. Insurance claim experts. Free inspection — (806) 622-6041"` (under 160 chars). (1 page)
- **Trim over-160-char descriptions:** For descriptions over 160 characters, truncate at the last complete sentence that fits under 160, or cut the last clause before the period. Ensure the result still reads naturally and retains the CTA if one exists. If trimming removes the CTA, append a short one. (19 pages)
- **Add CTA to descriptions missing one:** If description does not contain any of: "call", "Call", "free", "Free", "schedule", "Schedule", "(806)", "622-6041", then append `. Call (806) 622-6041` or `. Free estimate — (806) 622-6041`. Choose whichever fits under 160 chars. (17 pages)
- **Pad very short descriptions:** If description is under 80 characters, it's too thin. Append city-specific detail + CTA to reach ~120-150 chars. (1 page: /reviews/)
- **Normalize quotes:** If description uses single quotes and contains no apostrophes, convert to double quotes for consistency. If it uses single quotes WITH apostrophes, convert to double quotes (which safely contain apostrophes).
- **Update OG descriptions:** For the 7 pages with openGraph.description, update to match the new description.

**Anti-patterns to enforce (per content-voice.md):**
- No description starting with "Looking for..."
- No empty claims like "quality service, affordable prices"
- Descriptions should be outcome-focused, not question-focused

**Modes:**
- `--dry-run`: Print all proposed changes without writing. Show before/after with char counts.
- `--report`: Output summary table of all descriptions with char counts and issue flags.
- No flag: Apply changes and write files.

**Output:** Print counts: total files, descriptions fixed, already correct, needing manual review.

Run with `--dry-run` and review output. Verify no description exceeds 160 chars, all start uppercase, all have a CTA.
