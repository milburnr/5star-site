<task type="auto">
  <name>Task 2: Create internal link checker script</name>
  <files>scripts/audit-internal-links.js</files>
  <action>
    Create `scripts/audit-internal-links.js` — a Node.js script that:

    1. Recursively finds all `.html` files in `out/`
    2. Extracts all `href="/..."` values from each HTML file using regex
    3. Strips query strings (`?...`) and hash fragments (`#...`) from each href
    4. Skips non-path hrefs (external URLs, `tel:`, `mailto:`, `javascript:`)
    5. For each internal href, checks that the target exists in `out/`:
       - If href ends with `/`, check `out/{href}/index.html` exists
       - If href does not end with `/`, check `out/{href}` exists OR `out/{href}/index.html` exists
       - Special case: `/` resolves to `out/index.html`
    6. Collects all broken links with source file and target href
    7. At the end, prints summary:
       - Total pages scanned
       - Total internal links checked
       - PASS (exit 0) if zero broken, or FAIL (exit 1) with grouped output showing each broken href and how many pages reference it

    Follow the pattern from the research file code example. Place in `scripts/` directory alongside other audit scripts. Use `require('fs')` and `require('path')` only — no external dependencies.

    Run against current build to verify it works:
    ```
    node scripts/audit-internal-links.js
    ```

    Do NOT fix any broken links found — just confirm the script works correctly. Broken link fixes are Plan 10-03 scope.
  </action>
  <verify>

    ```
    node scripts/audit-internal-links.js
    ```
    Script runs, scans all HTML files in out/, and reports either PASS or a list of broken links. Exit code is 0 or 1 appropriately.
