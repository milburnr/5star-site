<task type="auto">
  <name>Task 2: Write and run script to add page-specific OG overrides to all pages</name>
  <files>scripts/add-og-tags.js</files>
  <action>
Create a Node.js script at `scripts/add-og-tags.js` that adds page-specific `openGraph` blocks to every page.tsx that has its own metadata export but no openGraph.

The script should:

1. Recursively find all `page.tsx` files under `app/`
2. For each file, check if it has `export const metadata` or `export const metadata:`
3. Skip files that already have `openGraph` in their metadata
4. For files that have metadata but no openGraph, extract:
   - `title` value from the metadata
   - `description` value from the metadata
   - The page slug from the directory path (e.g., `app/roof-repair-amarillo/` → `roof-repair-amarillo`)
5. Insert an `openGraph` block into the metadata object, right after the `description` line:

```typescript
  openGraph: {
    title: "{extracted_title}",
    description: "{extracted_description}",
    url: "https://5starroofingpros.com/{slug}/",
    siteName: "5 Star Roofing",
    images: [
      {
        url: "https://pub-797574ea9b1b4ccda73d4f6afb5d90d5.r2.dev/images/completed/completed-amarillo-4-1280w.jpg",
        width: 1280,
        height: 720,
        alt: "5 Star Roofing - Professional Roofing Services in Amarillo, TX",
      },
    ],
    locale: "en_US",
    type: "website",
  },
```

For the homepage (`app/page.tsx`), use `url: "https://5starroofingpros.com/"` (no slug).

The script should:
- Support `--dry-run` flag
- Log count of files modified, files skipped (already have OG), files unchanged
- Handle edge cases: multiline description strings, template literals in metadata

Run:
```bash
node scripts/add-og-tags.js --dry-run   # Preview
node scripts/add-og-tags.js              # Execute
```

After running, verify:
```bash

# Expected: 200+ (all pages with metadata)
```
