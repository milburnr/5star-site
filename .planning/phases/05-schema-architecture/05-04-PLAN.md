<task type="auto">
  <name>Task 1: Build and run schema validation script</name>
  <files>scripts/validate-schema.js</files>
  <action>
Create `scripts/validate-schema.js` — a Node.js script that:

1. **Reads all HTML files** from `out/` directory (glob `out/**/*.html`)
2. **Extracts JSON-LD blocks** from each file (regex for `<script type="application/ld+json">...</script>`)
3. **Parses each block** with JSON.parse (report parse errors)

4. **Validates these rules per page:**

   **Rule 1: LocalBusiness @type must be array**
   - If any schema block has `"@type": "LocalBusiness"` (string), ERROR
   - Expected: `["LocalBusiness", "RoofingContractor"]`
   - Exception: none

   **Rule 2: Entity name consistency**
   - If any schema block has `"name": "5 Star Commercial Roofing"` where @type is LocalBusiness/RoofingContractor/Service, ERROR
   - Expected: `"5 Star Roofing"`
   - Exception: Blog and Gallery page schemas may use descriptive names (CollectionPage, Blog @type)

   **Rule 3: WebSite schema on homepage**
   - `out/index.html` must contain a block with `"@type": "WebSite"`, ERROR if missing
   - Must have `sameAs` array with at least 1 URL

   **Rule 4: Person schema on about page**
   - `out/about/index.html` must contain a block with `"@type": "Person"`, ERROR if missing
   - Must have `worksFor` property

   **Rule 5: FAQ content ↔ FAQ schema consistency**
   - If an HTML file contains `AccordionTrigger` or FAQ-like accordion markup but has no `"@type": "FAQPage"` schema, WARNING
   - If a file has `"@type": "FAQPage"` but no accordion markup, WARNING (possible false schema)

   **Rule 6: Valid JSON-LD @context**
   - Every schema block must have `"@context": "https://schema.org"`, ERROR if missing or different

   **Rule 7: BreadcrumbList presence**
   - Every page except `out/index.html` should have BreadcrumbList schema, WARNING if missing

5. **Output format:**
   ```
   Schema Validation Report
   ========================
   Pages scanned: NNN
   Schema blocks parsed: NNN

   ERRORS: N
     - out/some-page/index.html: LocalBusiness @type is string, not array
     - ...

   WARNINGS: N
     - out/some-page/index.html: FAQ content found but no FAQPage schema
     - ...

   PASS / FAIL
   ```

6. **Exit code:** 0 if zero errors (warnings OK), 1 if any errors

7. **Run the script:**
   ```bash
   npm run build  # Build first (if not already built from 05-01/02/03)
   node scripts/validate-schema.js
   ```

8. **If errors are found:** Fix them. The most likely issues:
   - A few pages missed by the mass update script (fix manually or re-run)
   - JSON parse errors from malformed schema (fix the source page.tsx)

9. **Add npm script** to package.json: `"validate:schema": "node scripts/validate-schema.js"`
