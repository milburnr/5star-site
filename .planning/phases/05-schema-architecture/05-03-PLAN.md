<task type="auto">
  <name>Task 1: Identify FAQ gap pages and add FAQPage schema</name>
  <files>app/**/page.tsx (~34 files)</files>
  <action>
Write a Node.js script (scripts/fix-faq-schema-gaps.js) that:

1. **Identify gap pages:** Find all `app/**/page.tsx` files that contain `AccordionTrigger` (FAQ UI) but do NOT contain `"FAQPage"` (schema). Print the list of gap pages.

2. **Extract FAQ content:** For each gap page, parse the AccordionTrigger/AccordionContent pairs to extract question/answer text. The pattern in these files is:
   ```tsx
   <AccordionTrigger>Question text here?</AccordionTrigger>
   <AccordionContent>Answer text here.</AccordionContent>
   ```
   Use regex to extract Q&A pairs. Clean HTML tags from the extracted text (strip `<p>`, `<strong>`, etc. leaving plain text).

3. **Generate FAQPage schema:** For each gap page, generate a FAQPage JSON-LD block:
   ```json
   {
     "@context": "https://schema.org",
     "@type": "FAQPage",
     "mainEntity": [
       {
         "@type": "Question",
         "name": "Question text?",
         "acceptedAnswer": {
           "@type": "Answer",
           "text": "Answer text."
         }
       }
     ]
   }
   ```

4. **Insert into page:** Add a new `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(faqSchema) }} />` block. Insert it right BEFORE the closing `</Head>` or right after the last existing `application/ld+json` script tag in the file. Follow the pattern used by existing pages that have FAQ schema.

5. **Support --dry-run** to preview which pages would be modified and how many Q&A pairs extracted.

Run with --dry-run first, then for real. Print summary: N pages fixed, M total FAQ items added.

IMPORTANT: Some AccordionTrigger usage may not be FAQ content (e.g., service detail accordions). The script should log which pages and Q&A pairs it extracts for manual verification. If a page has fewer than 2 Q&A pairs or the content looks non-FAQ (no question marks in triggers), skip it and log a warning.
  </action>
  <verify>
   - `grep -rl 'AccordionTrigger' app/ | wc -l` = `grep -rl 'FAQPage' app/ | wc -l` (or very close — all FAQ pages have schema)
   - Spot-check 3 previously-gap pages: they now have FAQPage schema with correct Q&A text
   - `npm run build` succeeds
  </verify>
  <done>All pages with FAQ accordion content now have corresponding FAQPage schema. Gap reduced from ~34 to 0.</done>
</task>
