Standardize the business entity name, email, phone format, hours, and copyright year across all 246 pages and schema files so every page presents identical NAP (Name, Address, Phone) data matching constants.ts.

Inconsistent NAP data (3 phone formats, 2 email addresses, 2 hours values, entity name variations with city/service suffixes) prevents Google from building a confident entity profile, suppressing local pack and organic rankings.

Output: All pages standardized to canonical NAP values from constants.ts. Entity name consistent. Copyright confirmed at 2026.
</objective>

  <action>
Create a Node.js script at `scripts/standardize-nap.js` that processes all `page.tsx` files under `app/`. The script reads each file as a string and applies these replacements (in order):

**1. Email standardization** (COMP-03):
```javascript
// Replace info@ with admin@ in schema email fields
content = content.replace(/"email"\s*:\s*"info@5starroofingpros\.com"/g, '"email": "admin@5starroofingpros.com"');
```

**2. Phone format standardization** (COMP-04):
```javascript
// Normalize all phone formats in schema to match constants.ts
content = content.replace(/"telephone"\s*:\s*"\+18066226041"/g, '"telephone": "(806) 622-6041"');
content = content.replace(/"telephone"\s*:\s*"\+1-806-622-6041"/g, '"telephone": "(806) 622-6041"');
// Also catch any E.164 format without country code
content = content.replace(/"telephone"\s*:\s*"8066226041"/g, '"telephone": "(806) 622-6041"');
```

**3. Hours standardization** (COMP-04):
```javascript
// Fix 24/7 hours to match constants.ts business hours (9 AM - 5 PM)
content = content.replace(/"opens"\s*:\s*"00:00"/g, '"opens": "09:00"');
content = content.replace(/"closes"\s*:\s*"23:59"/g, '"closes": "17:00"');
```

**4. Entity name in schema 'name' field** (COMP-02):
The schema `"name"` field currently has variations like:
- `"5 Star Commercial Roofing - Amarillo Roof Repair"`
- `"5 Star Commercial Roofing - Midland"`
- `"5 Star Commercial Roofing"`

Per constants.ts, the canonical business name is `"5 Star Commercial Roofing"`. For the schema `"name"` property specifically, standardize to the plain business name WITHOUT city/service suffixes:
```javascript
// Remove city/service suffixes from schema business name
// Match "5 Star Commercial Roofing - anything" and replace with just "5 Star Commercial Roofing"
// ONLY in schema "name" property context (preceded by "name":)
content = content.replace(/"name"\s*:\s*"5 Star Commercial Roofing\s*-\s*[^"]+"/g, '"name": "5 Star Commercial Roofing"');
```

**IMPORTANT: Do NOT modify:**
- Meta titles (`<title>` tags or `metadata.title`) -- those can keep "5 Star Roofing" variant and city names for SEO
- OG tags -- those will be addressed in Phase 3
- The `name` property inside `Service` schema (which describes the SERVICE name, not the business)
- Any text content or headings

The script should:
- Support `--dry-run` flag for preview
- Log counts for each replacement type
- Report files with zero changes (for audit)

Run:
```bash
node scripts/standardize-nap.js --dry-run   # Preview
node scripts/standardize-nap.js              # Execute
```
