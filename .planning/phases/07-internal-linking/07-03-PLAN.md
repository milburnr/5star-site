  <action>
For each of the ~147 spoke pages that import InternalLinks but don't render it, add the JSX call.

**Strategy:** Write a Node.js script (temporary, delete after use) that:
1. Scans all `app/*/page.tsx` files
2. Identifies files that have `import { InternalLinks }` but no `<InternalLinks` in the JSX
3. For each file, determines the currentCity and currentService from the directory name pattern `/{service}-{city}/`
4. Inserts `<InternalLinks currentCity="{city}" currentService="{service}" />` before the final CTA section

**Placement rule:** Insert the InternalLinks component BEFORE the final CTA section (the `<section className="bg-gradient-to-r from-brand-brown to-brand-gold"` block). This matches the pattern used by the 7 pages that already render it (see hail-damage-repair-borger as reference).

**Service-to-slug mapping for parsing directory names:**
- residential-roofing-{city} -> service="residential-roofing", city="{city}"
- commercial-roofing-{city} -> service="commercial-roofing", city="{city}"
- hail-damage-repair-{city} -> service="hail-damage-repair", city="{city}"
- storm-damage-repair-{city} -> service="storm-damage-repair", city="{city}"
- wind-damage-repair-{city} -> service="wind-damage-repair", city="{city}"
- roof-replacement-{city} -> service="roof-replacement", city="{city}"
- roof-repair-{city} -> service="roof-repair", city="{city}"
- roof-inspections-{city} -> service="roof-inspections", city="{city}"
- tpo-roofing-{city} -> service="tpo-roofing", city="{city}"
- metal-roofing-{city} -> service="metal-roofing", city="{city}"
- asphalt-shingle-roofing-{city} -> service="asphalt-shingle-roofing", city="{city}"

The script should parse the directory name by matching against known service prefixes (longest match first to handle "asphalt-shingle-roofing" vs "roofing") and extracting the city slug as the remainder.

**After running the script:**
- Delete the script file
- Run `npm run build` to verify all 147 modifications compile
